\org\projasource\httpclient

\org\projasource\httpclientHttpRequest

Starting point for making a http requests.

Uses builder pattern making your code easy to write, and easy to understand.

A basic example of usage would be:


$response = HttpRequest::build("https://example.com")
    ->method("POST")
    ->contentType("application/json")
    ->entity(json_encode($entity))
    ->proxy("http://user:password@192.168.0.1:8080")
    ->accept("application/json")
    ->execute();

echo $response->getEntity()->field;

entity()

entity(mixed $entity) : \org\projasource\httpclient\HttpRequest

Set's an entity to be used in HttpRequest.

Could be of any type, conversion will be made using when sending a request with a {@link contentType($type)} set.

If a content type, or type of entity are not recognized, conversion to string will be obtained.

If request type does not asume any entity (GET, HEAD), it will be ignored.

contentType()

contentType(string $type) : \org\projasource\httpclient\HttpRequest

A content type of the entity.

Asumes a format of [type "/" subtype], as described in RFC 2615 section 3.6.1.

Content type is further used to convert entity into string (if necessary) while sending a request body, and is set into 'Content-Type' header.

NOTICE! Any data after semicolon (";") will be lost.
